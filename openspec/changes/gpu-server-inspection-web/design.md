# Design: GPU Server Inspection Web Tool

## Context

运维需要在**本地**集中查看多台**远程** GPU 服务器的 NUMA 拓扑、固件版本与 GPU 指标，且**不能**在 GPU 服务器上部署任何采集脚本或 Agent。本工具在用户本机运行后端 + 前端，通过 SSH 登录远程 GPU 服务器并执行既有命令（nvidia-smi、numactl、lspci 等），将输出取回解析后通过 API 与 Web 界面展示。主机列表及登录凭证通过**导入 Excel 表**加载（IP、账号、密码）。约束：远程主机需开放 SSH、已安装上述命令；本机仅需 Python 与依赖，无需 GPU 或 nvidia 驱动。

## Goals / Non-Goals

**Goals:**

- 本机运行，浏览器操作：导入 Excel 加载 GPU 主机（IP + 账号密码），选择主机后查询 NUMA 拓扑、固件/版本、GPU 指标，并支持一键巡检与汇总。
- 不在远程 GPU 服务器上安装或部署任何脚本或常驻进程；仅通过 SSH 执行已有系统命令并取回结果。
- 前后端分离：后端提供 REST API（含主机列表、按主机查询、巡检），前端通过 API 展示并支持 Excel 上传。

**Non-Goals:**

- 不实现完整的用户登录/权限体系（首版假定本机与内网可信）。
- 不替代专业监控系统（如 Prometheus/Grafana）；不做历史数据存储与趋势分析。
- 不在远程主机上安装 Agent、不要求远程预装本工具。

## Decisions

1. **后端技术选型：Python + FastAPI**
   - 理由：SSH 库（paramiko）成熟、命令解析简单；FastAPI 适合快速提供 REST API 与文件上传（Excel）。本机无需 GPU 环境。
   - 备选：Node.js（ssh2）—— 生态上 Python 在运维脚本与 Excel 解析更常见。

2. **远程采集方式：SSH 执行命令**
   - 理由：不向远程部署脚本；通过 SSH 连接后执行 `nvidia-smi --query-*`、`numactl -H`、`lspci -v` 等，将 stdout 取回本机解析。使用 paramiko 或 asyncssh，支持密码认证（Excel 中提供密码）。
   - 备选：在远程部署 Agent 再拉取 —— 与“不部署到 GPU 上”冲突，不采纳。

3. **主机与凭证来源：Excel 导入**
   - 理由：用户通过 Web 界面上传 Excel 文件，后端解析得到主机列表（至少：IP、用户名、密码）；可选列：SSH 端口、备注。导入后列表保存在内存或本地轻量存储（如 JSON 文件），供后续“选择主机 → 查询”使用。不要求用户在本机手工配置 SSH config。
   - Excel 格式：首行表头，列名约定为如 `IP`/`主机`、`用户名`/`账号`、`密码` 等（可文档约定或支持多语言列名映射）。
   - 备选：手动在界面添加单条主机 —— 可作为增强，首版以 Excel 批量导入为主。

4. **按主机查询的 API 形态**
   - 所有“查询远程信息”的 API（NUMA、版本、GPU 指标、巡检）均接受“主机标识”（如 IP 或导入后的主机 id）；后端根据该标识从已加载列表中取凭证，建立 SSH 执行对应命令，返回该主机的数据。可选：支持批量查询多台主机（多请求或单请求多 id），首版可先支持单主机查询。

5. **前端技术选型：单页 HTML + 原生 JS**
   - 理由：首版以“小工具”为主；需支持文件上传（Excel）、主机列表展示、主机选择、各模块结果展示。可先用静态页 + fetch，必要时再引入轻量框架。
   - 前端需包含：导入 Excel 入口、已加载主机列表、选择当前主机、导航到拓扑/版本/指标/巡检并传当前主机 id。

6. **部署形态：本机单进程后端 + 静态前端**
   - 理由：用户在本机执行 `uvicorn` 启动后端，浏览器访问本机地址；前端由 FastAPI 挂载静态文件。所有连接由本机发起至远程 GPU 服务器，不在 GPU 上部署任何服务。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 远程 SSH 失败（网络、防火墙、凭证错误） | API 返回明确错误信息（连接失败、认证失败、命令执行超时）；前端展示 per-host 错误状态。 |
| Excel 中明文密码 | 文档说明仅限内网/可信环境；可选后续支持密钥认证或从安全存储读取。 |
| 解析远程命令输出受 locale/版本影响 | 尽量使用机器可读格式（如 nvidia-smi --query）；对关键字段做兼容或版本检测；解析失败时返回明确错误。 |
| 并发多主机查询时 SSH 连接数 | 首版可串行或限制并发；后续可加连接池与超时控制。 |

## Migration Plan

- **部署**：仅在本机安装 Python 3.8+ 与依赖（FastAPI、paramiko、openpyxl 等），运行后端；浏览器访问本机端口。无需在 GPU 服务器上做任何安装。
- **回滚**：停止本机服务即可；不影响远程 GPU 服务器。
